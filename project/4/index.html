從冰冷表單到溫暖對話

以下依期末簡報整理：先看見台灣 ACP 的結構性困境，再以設計思考五步驟，把「不敢談」收斂成可落地的 Flutter 原型。

背景 — 為什麼需要重新想 ACP

善終難關：不健康的餘命

台灣人平均壽命雖長，但在生命終點前，往往經歷漫長且痛苦的「無效醫療」折磨。氣切、插管、洗腎與葉克膜等激烈的加工急救措施，讓無數長輩求生不得、求死不能——臨終前平均 8.4 年失能臥床。

家庭決策的兩難

當事人陷入昏迷，決定權落在毫無心理準備的子女身上。「誰簽字，誰不孝」——為了向親戚交代，被迫選擇繼續加工急救。

理想很豐滿，現實很骨感

台灣 ACP 簽署率僅約 1%。自費諮商費用高昂、法規流程生硬，推動遠比想像困難。

四重阻力疊加

低簽署率不全是「不知道有這條法」——而是死亡禁忌 × 家庭決策文化 × 冰冷法律語言 × 高齡數位落差，同時壓在長者與家屬身上。

問題重新定義

我們想重新定義問題

一般團隊可能會做「加速行政流程、提高簽署率」的數位化——但這真的是我們要解決的核心痛點嗎？

我們的答案是：不是。長輩需要的不是另一份文件，而是一個能「先想想、再開口、最後和家人談」的低焦慮起點。

設計思考流程

Design Thinking

依 Stanford d.school 五步驟推進：同理 → 定義 → 構思 → 原型 → 測試。每一步都對應簡報中的研究洞察與功能收斂。

  1. 同理 Empathize — 訪談羅醫師，整理四項核心洞察。
  2. 定義 Define — 從「簽署工具」轉為「溝通工具」。
  3. 構思 Ideate — 以 COM-B、TTM、SCT 收斂功能。
  4. 原型 Prototype — Flutter Android MVP，聚焦溝通與心理準備。
  5. 測試 Test — 原型驗證完成，規劃語音、LLM 與實測迭代。
Step 1 — 同理心 Empathize

我們聽到了什麼？

訪談安寧專家羅玉岱醫師後，我們整理出四項洞察，也看見長者在數位介面上的真實阻力。

  1. 洞察 1：死亡是禁忌，很多人連想都不願想。
  2. 洞察 2：每個人所在「階段」不同，不能一套衛教打天下。
  3. 洞察 3：APP 的角色是「點火苗」，不是教人簽文件。
  4. 洞察 4：台灣以「家庭」為決策單位。

極深的文化禁忌

直接談「死亡」「預立」會引發強烈抗拒。

排斥複雜介面

小字、多步驟表單，對長者幾乎不可用。

害怕承擔決策責任

長者怕連累子女；子女怕背負「不孝」標籤。

Step 2 — 定義問題 Define

溝通工具，不是簽署工具

如果把 ACP 紙本表單照抄到手機上，那不叫設計，那叫數位化。我們把問題重新定義如下：

簽署工具 ❌

「如何讓長者快速填寫並簽署 ACP 表單？」

溝通工具 ✓

「如何為長者與家人創造一個降低焦慮、順應心理、且能促進家庭對話的溝通工具？」

Step 3 — 創意發想 Ideate

如何把創意收斂成能落地的功能？

我們整合 COM-B 模型、跨理論模式（TTM）與社會認知理論（SCT），用三個問題檢核每個功能：「卡在哪？現在準備到哪個階段？用什麼設計讓人願意動？」

COM-B：能力 × 機會 × 動機

維度 ACP 障礙 功能發想
能力 Capability 不會開口、不會用手機、用語太難 語音互動、大字二選一、台語 TTS
機會 Opportunity 諮商貴、要家人陪、不知去哪諮商 家屬模式、諮商連結、低門檻入口
動機 Motivation 避諱死亡、怕不孝、怕等於放棄治療 「陪你想想」故事、提前想≠現在決定

TTM：依準備階段給不同內容

階段 心理目標 專屬內容 / 主力功能
懵懂期 降低抗拒 故事 + 影片 · 聽故事、點燃火苗
思考期 釐清價值觀 價值引導 · 想想看 → 心願小卡
準備期 學會開口 開場話術 · 家屬陪伴模式
行動期 銜接正式程序 諮商指引 · 查詢 ACP 諮商機構

設計原則：不強迫、沒標準答案、聊一點是一點、不等於放棄治療。

SCT：用別人的故事，想自己的事

  • 觀察學習：王伯伯／阿秀姨互動故事、阿嬌姨影片——「用別人的故事，想自己的事」。
  • 自我效能：家屬陪伴開場話、想想看一步步選、語音優先降低操作難度。
  • 結果期待：文案強調「提前想 ≠ 現在決定」「是愛的表現」「家人不用在最慌亂時猜」。
Step 4 — 製作原型 Prototype

「陪你想想」MVP

以 Flutter 開發 Android App 原型，幫助高齡者與家屬理解與準備 ACP——聚焦溝通與心理準備，不是完整簽署系統。

做了什麼

  • 2～4 題階段評估 + 分期專屬內容
  • 互動故事、想想看、心願小卡
  • 家屬陪伴模式 + 語音 FAQ（STT + 關鍵字比對）
  • 台語 TTS、大字介面、長按確認互動

刻意不做

  • 線上簽署 AD
  • 完整病歷或雲端資料
  • LLM 開放式聊天

四大模組

模組 功能內容 解決什麼
① 階段評估與分期內容 2～4 題評估時期，各期專屬文案、影片及引導 不是人人都準備好，不能一套內容用到尾
② 互動引導 聽故事（王伯伯、阿秀姨）、想想看、心願小卡 避諱、不會表達、想法模糊
③ 家屬陪伴模式 長輩開場話、家屬聆聽提示、「一起聊聊」模擬對話 想說不會說、家人聽不進去
④ 語音互動 台語 TTS 朗讀、語音 FAQ（STT + 關鍵字比對） 視力差、討厭打字、用語太難

行為階段評估（2～4 題）

透過簡短提問替使用者做 TTM 分類，再對應後續功能。交互設計：點擊播放語音、長按確認選項。

階段 專屬內容 主推功能
懵懂期聽一個故事、相關案例與影片影片觀看 · 點燃火苗
思考期價值觀引導（什麼對我最重要）想想看 → 心願小卡
準備期與家人互動的引導、釐清不清楚的內容家屬陪伴模式 · 問答模式
行動期正式諮商說明查詢 ACP 諮商機構

聽一個故事

用別人的故事讓長輩想自己的事——不說教，透過情境選擇慢慢點出「提前想一想」的意義。

心願小卡

把「想想看」的選擇整理成「我希望……」的句子，給家人、給醫師、給不敢開口的自己。

家屬陪伴模式

長輩和家人輪流說話，App 一步步帶，不用自己想怎麼開場。

Step 5 — 測試與迭代 Test

已完成的驗證與未來方向

已完成

  • Flutter Android App 可互動原型
  • TTM 分期內容 + 語音 FAQ 技術驗證
  • 高齡友善 UI 與家屬模式骨架

未來迭代

  • 優化語音：引進自然語調，消除機械音
  • LLM 語境受控導入：安全探討 ACP 核心問答
  • 家屬陪伴實測：在真實情境中驗證溝通效果
  • 量化統計數據以支持國科會後續學術研究
